# Shared test vectors

Language-agnostic JSON fixtures used to keep the two engine implementations in
sync:

- **C#** — `backend/src/Solitaire.Engine` (authoritative; generated + verified by
  `Solitaire.Engine.Tests`)
- **TypeScript** — the frontend engine (to be added)

Each vector is a **valid move sequence**. Replaying `{seed, options, moves}`
through an engine must reproduce `expectedFinalScore` and `expectedWin` exactly.
This is how we guarantee the TypeScript port stays bit-for-bit compatible with the
authoritative C# engine.

## Layout

```
shared/
  vectors/
    klondike.json     # generated by Solitaire.Engine.Tests — do not hand-edit
```

`klondike.json` is regenerated by the `EmitVectorsFile` test and kept in sync by
the `CommittedFile_IsInSyncWithGenerator` test, so it always matches the current
engine behavior.

## File format

```jsonc
{
  "description": "…",
  "vectors": [
    {
      "name": "draw1-seed2-win",
      "seed": 2,                       // integer seed for the deterministic shuffle
      "options": {
        "drawCount": 1,                // 1 or 3
        "maxRedeals": 2147483647       // int.MaxValue == unlimited
      },
      "moves": [                       // applied in order; each must be legal
        { "type": "TableauToFoundation", "source": 6 },
        { "type": "Draw" },
        { "type": "WasteToTableau", "destination": 5 },
        { "type": "TableauToTableau", "source": 3, "destination": 0, "count": 2 }
        // …
      ],
      "expectedFinalScore": 510,
      "expectedWin": true
    }
  ]
}
```

### Move types and their fields

| `type`                | fields used            | meaning |
| --------------------- | ---------------------- | ------- |
| `Draw`                | –                      | flip `drawCount` cards stock → waste |
| `Recycle`             | –                      | waste → stock (a redeal) |
| `WasteToFoundation`   | –                      | waste top → its suit foundation |
| `WasteToTableau`      | `destination`          | waste top → tableau pile 0..6 |
| `TableauToFoundation` | `source`               | pile `source` top → its suit foundation |
| `FoundationToTableau` | `source`, `destination`| foundation of suit `source` (0..3) → tableau |
| `TableauToTableau`    | `source`, `destination`, `count` | move a `count`-card run between piles |

Omitted fields default to "not applicable". The deterministic shuffle, deal, and
scoring rules are documented in the C# engine (`Deck`, `DeterministicRandom`,
`Scoring`) and must be reproduced identically by any port.
